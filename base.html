Tag <base>

A tag que define a URL base para todos os links relativos em uma página.

Definição

A tag <base> especifica a URL base a ser usada para todos os URLs relativos em um documento. Só pode haver um elemento <base> em um documento, e ele deve estar dentro do elemento <head>.

Isso é útil para centralizar os caminhos, facilitando a manutenção de links e recursos (como imagens e scripts) quando a estrutura de pastas do site muda.

Exemplo de Uso

No exemplo abaixo, a URL base é definida como "https://developer.mozilla.org/". Todos os links relativos na página serão resolvidos a partir dessa URL.

<!DOCTYPE html>
<html>
  <head>
    <base href="https://developer.mozilla.org/" target="_blank">
    <title>Exemplo de Base</title>
  </head>
  <body>
    <!-- Este link apontará para "https://developer.mozilla.org/pt-BR/docs/Web/HTML" -->
    <a href="pt-BR/docs/Web/HTML">Documentação de HTML</a>

    <!-- Esta imagem será carregada de "https://developer.mozilla.org/static/img/favicon32.png" -->
    <img src="static/img/favicon32.png" alt="Logo MDN">
  </body>
</html>

Note que o atributo target="_blank" na tag <base> fará com que todos os links da página abram em uma nova aba por padrão.

Atributos

Atributo Valor Descrição
href URL (Obrigatório) Define a URL base para todos os caminhos relativos na página. Deve ser uma URL absoluta.
target _self, _blank, _parent, _top Define o contexto de navegação padrão para links e formulários que não possuem um atributo target explícito.

Boas Práticas e Observações

  • Posicionamento: Coloque a tag <base> o mais cedo possível dentro do <head>. Qualquer recurso com URL relativa (como <link rel="stylesheet">) declarado antes da tag <base> não será afetado por ela.
  • Apenas uma por página: Um documento HTML só pode ter uma única tag <base>.
  • Cuidado com Âncoras: Links de âncora (href="#secao") também são afetados pela tag base. Se você tem <base href="/docs/">, um link para #secao apontará para /docs/#secao, o que pode quebrar a navegação interna da página.

Considerações de Acessibilidade

O uso de target="_blank" na tag <base> força a abertura de todos os links em uma nova aba, o que pode ser desorientador para usuários, especialmente aqueles que utilizam leitores de tela. Se for utilizá-lo, é uma boa prática avisar visualmente (com um ícone) e textualmente (para leitores de tela) que o link abrirá em uma nova janela.
